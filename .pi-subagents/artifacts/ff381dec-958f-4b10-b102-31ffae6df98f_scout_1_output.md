# JetBrainsMono Nerd Font 3.4.0 animation-family audit

## Scope and method

Inspected Nerd Fonts upstream `glyphnames.json` **v3.4.0** (`METADATA.date` 2025-04-24) and the installed terminal face:
`/nix/store/xfgcrc59dijia0wqxv5lxfpg1234mipg-nerd-fonts-jetbrains-mono-3.4.0+2.304/share/fonts/truetype/NerdFonts/JetBrainsMono/JetBrainsMonoNerdFontMono-Regular.ttf`.
A Python/fontTools pass grouped every metadata name by trailing decimal, word-number (`one`…`twelve`), direction, and common state token; each reported code point was then looked up in the font's merged cmap and measured through `hmtx` and outline bounds. **Every inventory item below is cmap-present and has advance 600 font units (one cell).** “bbox identical” is exact outline-bound equality; “varies” is not necessarily unsafe, only a possible apparent wobble. All 10,764 metadata entries are present in the Mono face cmap (which contains 11,792 mapped code points). PUA characters have no portable Unicode width semantics; the font's 600-unit advance is the meaningful verification.

## Ranked strong candidates

1. **`extra-progress_spinner_1`…`_6` (U+EE06–U+EE0B)** — purpose-built six-frame spinner; cyclic motion; best semantic fit. Advance stable, bbox varies (intentional moving stroke). Pair with static progress caps `extra-progress_empty_left` U+EE00, `extra-progress_empty_mid` U+EE01, `extra-progress_empty_right` U+EE02 and full U+EE03/U+EE04/U+EE05 only for a multi-cell bar, not a one-cell spinner.
2. **`md-circle_slice_1`…`_8` (U+F0A9E–U+F0AA5)** — eight monotonic radial-fill frames; exact bbox and advance stability; excellent loop if 8→1 discontinuity is acceptable, excellent progress indicator otherwise.
3. **`weather-time_1`…`_12` plus `_12` (U+E381–U+E38C)** / **`md-clock_time_one`…`twelve` (U+F143F–U+F144A)** — 12-frame clock-hand rotation; exact bbox; reorder numerically (metadata/codepoint order begins at 12 for weather).
4. **Moon phase families** (eight 6-frame half-cycle families below) — smooth waxing/waning state progression, generally exact bbox; concatenate crescent→gibbous with the separately named quarter/full/new moon icons only after visual testing because naming does not expose one single contiguous cycle.
5. **`md-hexagon_slice_1`…`_6`** — six-frame geometric fill, exact bbox; strong deterministic progress animation.
6. **Battery/charging series** — 5–10 coherent level frames, exact bbox. Strong progress/status animation, but sort by suffix value, never code point; ordinary `md-battery` omits 0 and 100 named numeric members.
7. **Direction quartets** (`md-arrow`, arrow-circle, chevrons, weather direction/wind) — coherent rotation/navigation frames. `md-arrow` has exact bbox; most others vary. Direction order must be explicitly chosen (e.g. up→right→down→left), not codepoint order.
8. **Signal/Wi-Fi/network strength and lightbulb levels** — 3–9 exact-bbox growth frames; good pulse/progress, less naturally cyclic.
9. **Beaufort 0–12 / tally 1–5 / thermometers** — monotonic intensity/count animations. Beaufort/tally bboxes vary; thermometers are stable.
10. **Dice families** — coherent named sequences but semantic randomization rather than motion; useful “rolling” animation with shuffled rather than numeric playback.

## Explicit state endpoints worth combining

All are cmap-present/600 advance, but are two-state transitions rather than rich animation: `fa-toggle_off` U+F204 / `fa-toggle_on` U+F205; `iec-power_off` U+23FB / `iec-power_on` U+23FD; `md-toggle_switch_off` U+F0521 / `md-toggle_switch_on` U+F0522; `md-play_pause` U+F040E / `md-play` U+F040A; `md-door_closed` U+F081B / `md-door_open` U+F081C; `fa-folder` U+F07B / `fa-folder_open` U+F07C; `md-eye_closed` U+F0D6F / `md-eye` U+F0208; `md-volume_off` U+F0581, `md-volume_low` U+F057F, `md-volume_medium` U+F0580, `md-volume_high` U+F057E; `fa-volume_off` U+F026, `fa-volume_down` U+F027, `fa-volume_up` U+F028. These are coherent toggles/pulses; only volume is a useful 3–4-frame magnitude sequence.

## Exclusions (systematic)

- **Numeric but not animation:** `fa-dice_d{6,20}`, `md-dice_d{4,6,8,10,12,20}` are die *types*; `md-ev_plug_ccs{1,2}`, `md-ev_plug_type{1,2}`, `md-format_header1…6`, `md-home_floor_{1,2,3,a}`, `md-keyboard_f1…f12`, `md-numeric_0…9`, `md-roman_numeral_1…7`, `md-surround_sound_2_0/1`, and `md-timer_3/10` are categories/labels, not successive visual states. `md-fast_forward_*` and `md-rewind_*` are durations (5/10/15/30/60), included only as plausible changing-speed indicators, not smooth frames.
- **Directional false positives:** layout/alignment/dock/border/pan/gesture, import/export, thumb/hand, upload/download, and hundreds of object-arrow families express commands or alternatives, not a single object rotating. Only visually generic arrow/chevron/triangle and weather-direction quartets were retained. Two-member rotate-left/right families encode direction of rotation, not frames of it.
- **State suffix false positives:** the broad pass found hundreds of `open/closed`, `on/off`, `plus/minus`, `alert`, `outline/fill`, etc. Most are UI variants whose silhouettes/details change discontinuously. Rich families above and the explicit toggle list are the only plausible temporal sequences. Aliases sharing code points (e.g. Font Awesome old/new names) are not additional frames.
- **Incomplete “progress” bars:** `extra-progress_*_{left,mid,right}` are spatial tiles for a three-cell bar. Cycling them in one cell causes edge motion/clipping and is excluded from ranked single-cell animation.
- **Codepoint adjacency is not sequence evidence.** Several families are nonnumeric in cmap order (battery charging, clocks), and unrelated adjacent PUA glyphs must never be swept into an animation.

## Renderer risks: Ghostty / Zed

- **High:** configuration must select **`JetBrainsMono Nerd Font Mono`**, not the unqualified or `Propo` family. Only the Mono variant contractually normalizes patched icons to one 600-unit cell. Font fallback can silently select another Nerd Font or emoji glyph with different metrics.
- **High:** Material Design code points above U+FFFF (for example U+F0A9E) require correct UTF-8 scalar encoding. Do not emit UTF-16 surrogate halves or shell `\uFFFF`-limited escapes; use `\U000F0A9E`/actual UTF-8.
- **Medium:** PUA East Asian width is ambiguous/nonstandard. Ghostty's grid ultimately uses cell width plus font shaping; Zed's terminal and editor text renderer may choose fallback/shape runs differently. Validate the exact UI surface, font family, size, and line height; the font's advance does not guarantee a library such as `wcwidth()` agrees.
- **Medium:** outlines commonly touch x=0/x=600 and some exceed the nominal cell (`cod-arrow_circle` to x=601; progress caps x=-30…630). Ghostty/Zed may clip at cells, antialias differently, or leave dirty edge pixels during rapid replacement. Prefer exact-bbox Material Design fill families; avoid progress caps in one cell.
- **Medium:** varying bboxes create perceived jitter despite fixed advance. Direction quartets also swap wide/short dimensions. Centering is font-outline-dependent; test at target DPI and avoid italic faces.
- **Low/medium:** ligatures and font features can reshape neighboring ASCII, though these single PUA scalars normally do not ligate. Keep each frame one scalar, avoid variation selectors/ZWJ, and redraw the same terminal cell rather than appending/backspacing. Zed editor animations are also constrained by its repaint cadence; terminal escape updates may be coalesced.
- **Low:** Glyph metadata names are not stored as a runtime API in the font; applications must embed the code points. Names can change across Nerd Fonts releases even when codepoints remain aliases. Pin 3.4.0 or validate at startup/deployment.

## Reproduction / validation

Commands used: `fc-list | grep -i JetBrainsMono.*Nerd`; `curl -L .../v3.4.0/glyphnames.json`; `nix-shell -p python3Packages.fonttools --run 'python /tmp/an.py'`; a second fontTools report pass; `git status --short`. The script read `cmap`, `hmtx`, and glyph outlines with `BoundsPen`. No project files were edited and no tests were added. Existing staged/worktree changes shown by git predated and are unrelated to this audit.
