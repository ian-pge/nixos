## Review
- **Correct:** Pairwise visible-state exclusivity is established after each show function completes. Volume, brightness, media, Wi‑Fi, Bluetooth, updates, and launcher all close competing widgets and stop the volume/brightness/media timers (`StatusData.qml:463-499`, `560-615`, `701-718`, `890-904`).
- **Correct:** Bounce removal is complete in audited central widgets: width/height and launcher/update content use monotonic `Easing.OutCubic`; no `OutBack`, spring, or overshoot remains (`Bar.qml:209-220`, `AppLauncher.qml:64-72`, `UpdateSelector.qml:30-38`).
- **Correct:** Same-size short-widget transitions retain stable 36px geometry, and interrupted width/height animations can retarget from their current value through standard QML `Behavior` animations (`Bar.qml:189-220`).

- **Medium:** During tall→short transitions, the incoming short widget fills the still-tall animated container (`Bar.qml:235-287`). Its vertically centered content therefore initially appears far below the bar and travels upward while height shrinks; this affects workspaces, volume, brightness, media, Wi‑Fi, and Bluetooth. Smallest fix: top-anchor each short component with `height: 36` and left/right anchors instead of `anchors.fill`.
- **Medium:** Cross-widget transitions bypass the full Wi‑Fi cleanup routine by assigning `wifiSelectorVisible = false` and only stopping scanning (`StatusData.qml:465-466`, `563-564`, `578-579`, `704-705`, `893-894`). Unlike `hideWifiSelector()` (`StatusData.qml:618-626`), this leaves `wifiConnectionTimer`, pending-network state, and password state alive. A timeout can later mutate hidden state, and reopening can retain password mode. Smallest fix: call `hideWifiSelector()` at those sites.
- **Medium:** A rapid close immediately after opening Wi‑Fi can resurrect scanning: `refreshWifiNetworks()` schedules an uncancellable `Qt.callLater`, whose callback enables scanning without checking visibility (`StatusData.qml:684-691`). Smallest fix: require `root.wifiSelectorVisible` in the callback, preferably with a request-generation token.
- **Medium:** Bluetooth closure does not clean an in-flight action. `hideBluetoothSelector()` only stops discovery (`StatusData.qml:721-725`), while `bluetoothActionTimer` continues for 20 seconds (`StatusData.qml:1072-1075`) and `bluetoothActionDevice !== null` blocks subsequent activation (`StatusData.qml:784-800`). Smallest fix: stop the action timer and clear action/device state when hiding, or explicitly preserve the operation while allowing a reopened selector to observe it.
- **Medium:** Wi‑Fi, Bluetooth, and updates target `Hyprland.focusedMonitor` (`StatusData.qml:609-610`, `712-713`, `901-902`) even when their pill was clicked on another monitor (`Bar.qml:134-153`). Only the launcher passes the clicked bar’s monitor (`Bar.qml:124-125`). Consequently keyboard exclusivity can be assigned to the previous focused monitor. Smallest fix: add optional target-monitor parameters to those toggles/show methods and pass `window.hyprlandMonitor.name`.

- **Note:** No runtime, frame-by-frame, monitor-hotplug, or asynchronous device-operation test was performed; findings are static-code attestation. Existing staged files predated this read-only audit.
